# Исправление отображения кириллицы в печатных формах

## Проблема
В печатных формах (PDF) кириллические символы отображались как квадратики вместо букв.

## Причина
ReportLab по умолчанию использует шрифт Helvetica, который не поддерживает кириллицу. Для корректного отображения русских букв необходимо использовать шрифты с поддержкой Unicode.

## Решение
Обновлен класс `PrintFormGenerator` для автоматической регистрации шрифтов с поддержкой кириллицы:

### Приоритет выбора шрифтов:
1. **Arial** (Windows) - основной вариант
2. **Times New Roman** (Windows)
3. **Calibri** (Windows)
4. **Verdana** (Windows)
5. **DejaVu Sans** (из папки проекта `fonts/`)
6. **Helvetica** (fallback, без поддержки кириллицы)

### Изменения в коде:
- Добавлен метод `_register_fonts()` для автоматической регистрации шрифтов
- Добавлен метод `_encode_text()` для корректной обработки текста
- Все методы создания текстовых элементов обновлены для использования кодирования

### Файлы изменены:
- `src/services/print_form_generator.py`

## Тестирование
Созданы тестовые скрипты для проверки:
- `test_cyrillic_fonts.py` - базовый тест кириллицы
- `test_print_forms.py` - тест печатных форм
- `test_print_forms_updated.py` - тест форм АРСД

Все тесты успешно пройдены. Кириллица отображается корректно.

## Результат
✓ Кириллические символы теперь отображаются корректно во всех печатных формах
✓ Используется шрифт Arial (встроен в Windows)
✓ Размер PDF файлов: ~85-90 KB (включая встроенный шрифт)

## Примечания
- Если Arial недоступен, система автоматически попробует другие шрифты
- Для использования DejaVu Sans поместите файлы шрифтов в папку `fonts/`
- При использовании Helvetica (fallback) кириллица не будет отображаться
