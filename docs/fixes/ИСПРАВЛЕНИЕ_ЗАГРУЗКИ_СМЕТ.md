# Исправление ошибки 422 при загрузке смет в ежедневном отчете

## Проблема

При открытии формы ежедневного отчета возникала ошибка 422:
```
GET /api/documents/estimates?page_size=1000
Ответ: 422 - "Input should be less than or equal to 100"
```

## Причина

API ограничивает максимальный размер страницы до 100 записей, но веб-клиент пытался загрузить 1000 записей за один запрос.

## Решение

Реализована автоматическая пагинация - данные загружаются через несколько запросов по 100 записей, пока не будут получены все данные.

## Исправленные компоненты

1. **Форма ежедневного отчета** - загрузка списка смет
2. **Форма табеля** - загрузка списка смет для фильтра
3. **Регистр выполнения работ** - загрузка смет и движений
4. **Справочник работ** - загрузка в иерархическом режиме
5. **Хранилище справочников** - загрузка всех справочников (контрагенты, объекты, работы, сотрудники, организации)

## Результат

✅ Ошибка 422 устранена
✅ Все данные загружаются корректно
✅ Веб-клиент собран и готов к использованию
✅ Файлы скопированы в папку build/web

## Проверка

После запуска приложения:
1. Откройте форму ежедневного отчета
2. В консоли браузера (F12) должны быть запросы с `page_size=100`
3. Список смет должен загрузиться без ошибок
4. Все сметы должны быть доступны для выбора

## Развертывание

Для применения исправлений на продакшн-сервере:
1. Скопируйте содержимое папки `build/web` на сервер
2. Перезапустите веб-сервер (если требуется)
3. Очистите кэш браузера (Ctrl+F5)
