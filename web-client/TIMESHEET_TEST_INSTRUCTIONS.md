# Инструкции по тестированию исправлений табеля

## Что было исправлено

1. **Выбор сметы при изменении объекта** - теперь список смет обновляется автоматически
2. **Горизонтальная прокрутка** - колонки дней месяца прокручиваются, а сотрудники и итоги остаются видимыми

## Как протестировать

### 1. Тест выбора сметы

1. Запустите веб-клиент: `npm run dev`
2. Войдите в систему
3. Перейдите в раздел "Документы" → "Табели"
4. Нажмите "Создать" или откройте существующий табель
5. Выберите объект из списка
6. **Проверьте:** список смет должен заполниться сметами для выбранного объекта
7. Измените объект на другой
8. **Проверьте:** список смет должен обновиться и показать сметы нового объекта
9. Поле "Смета" должно сброситься

### 2. Тест горизонтальной прокрутки

1. Откройте табель с данными (или создайте новый и добавьте сотрудников)
2. Уменьшите ширину окна браузера (или используйте режим мобильного устройства в DevTools)
3. **Проверьте:** 
   - Колонка "Сотрудник" и "Ставка" остаются видимыми слева
   - Колонки дней (1-31) прокручиваются горизонтально
   - Колонки "Итого", "Сумма" и "Действия" остаются видимыми справа
4. Прокрутите колонки дней вправо
5. **Проверьте:** можно увидеть все дни месяца и итоговые колонки

### 3. Визуальная проверка

- Выходные дни (суббота, воскресенье) должны быть выделены красным фоном
- Итоговые колонки должны иметь синий фон
- Строка "Итого" внизу должна быть выделена серым фоном

```bash
# Запуск dev-сервера
npm run dev

# Сборка для продакшена
npm run build

# Проверка типов

```

## Известные ограничения

- На очень узких экранах (< 320px) может потребоваться дополнительная прокрутка
- Фиксированные колонки занимают минимальную ширину для читаемости
